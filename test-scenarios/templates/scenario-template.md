## 📝 시나리오 작성 가이드

### **기본 구조**
- 각 시나리오는 단계별로 번호를 매긴다
- 전제조건은 첫 번째 단계에 포함한다
- 사용자 동작과 확인 사항을 명확히 구분한다

### **작성 원칙**
- **구체적인 값 사용**: "test@example.com", "password123!" 등
- **명확한 동작 표현**: "클릭한다", "입력한다", "확인한다"
- **예상 결과 명시**: "~가 표시되는지 확인한다"

### **예시**
```
1. 관리자 계정으로 로그인한다.
2. `/teams` 페이지로 이동한다.
3. "팀 생성" 버튼을 클릭한다.
4. "팀명" 입력란에 "QA 테스트 팀"을 입력한다.
5. "생성" 버튼을 클릭한다.
6. 새 팀이 목록에 표시되는지 확인한다.
```

### **주의사항**
- 복잡한 마크다운 문법 사용 금지
- 테이블이나 코드 블록 최소화
- 단순하고 읽기 쉬운 형태로 작성
